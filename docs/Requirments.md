# 项目需求文档：本地政务AI办事助手（V1.0 MVP）

**版本**：1.0
**最后更新**：2026年4月11日
**项目类型**：政务便民服务 / AI应用

------

## 1. 项目背景

市民在办理本地政务事项（社保、户籍、房产、企业开办等）时，普遍存在以下痛点：

- **不知道怎么办**：不清楚该去哪个部门、带什么材料
- **流程不清楚**：网上信息陈旧、相互矛盾
- **多头咨询**：需要打多个部门电话，经常被转接或得到不准确答复

现有通用AI（如DeepSeek、豆包）虽然能聊天，但缺乏本地政务知识、无法保证答案准确性、不能帮用户"办成事"。

**目标**：打造一个专注于本地政务咨询的AI助手，让市民"问得清、办得成、少跑腿"。

------

## 2. 用户画像

| 用户类型         | 特点                               | 核心需求                               |
| :--------------- | :--------------------------------- | :------------------------------------- |
| **普通市民**     | 对政务流程不熟悉，老年人居多       | 用"人话"提问，得到清晰、准确的办事指引 |
| **企业办事员**   | 需要办理营业执照、变更、补贴申请等 | 快速获取材料清单和线上办理入口         |
| **政务窗口人员** | 重复回答相同问题                   | AI分担咨询压力，减少重复劳动           |

------

## 3. 核心功能需求（MVP）

### 3.1 准确问答（P0 - 必须有）

- 基于本地政务知识库回答问题，**严禁编造**
- 答案必须注明来源（如"根据XX局2025年3月发布的办事指南"）
- 当无法确定答案时，主动提供官方咨询电话或转人工

### 3.2 模糊意图澄清（P0 - 必须有）

- 用户提问信息不全时，主动反问获取关键信息
- **示例**：
  - 用户："我想办社保"
  - AI："请问您是办社保卡、查社保记录、还是办社保转移？"

### 3.3 材料预审（P1 - 重要）

- 支持用户上传材料照片（身份证、申请表等）
- AI判断材料是否齐全、格式是否正确
- 给出明确的补充建议

### 3.4 精准导航（P1 - 重要）

- 告知用户具体的办事大厅名称、地址、办公时间
- 提供一键导航链接（高德/百度地图）
- 告知是否需要预约及预约入口

### 3.5 行动闭环（P2 - 锦上添花）

- 提供线上办理入口链接（政府APP/小程序）
- 支持查询办事进度（对接政务系统）

## 5. 技术架构建议

### 5.1 技术栈

| 层级 | 技术选型                             | 说明           |
| :--- | :----------------------------------- | :------------- |
| LLM  | DeepSeek-V3 / Qwen-14B               | 开源可本地部署 |
| RAG  | Chroma + BGE-large-zh                | 向量检索       |
| 框架 | LangChain（仅RAG部分）+ 自研对话管理 | 灵活可控       |
| OCR  | PaddleOCR                            | 材料预审       |
| 部署 | 8核32G + GPU（RTX 5060ti）           | 先本地跑       |

### 5.2 数据流

text

```
用户提问 → 敏感词过滤 → 意图识别 → 槽位追问（如信息不全）
    → RAG检索知识库 → LLM生成答案 → 答案审核 → 返回用户
```



### 5.3 知识库结构（示例）

text

```
knowledge_base/
├── 社保/
│   ├── 社保卡办理.txt
│   ├── 社保转移.txt
│   └── 灵活就业参保.txt
├── 户籍/
│   ├── 身份证补办.txt
│   ├── 户口迁移.txt
│   └── 居住证办理.txt
├── 房产/
│   ├── 房产证办理.txt
│   └── 二手房过户.txt
└── 企业/
    ├── 营业执照.txt
    └── 食品经营许可证.txt
```

## 6. 数据来源与维护

| 数据类型 | 来源                  | 更新频率        |
| :------- | :-------------------- | :-------------- |
| 办事指南 | 各局官网/政务大厅窗口 | 每月核对        |
| 政策文件 | 政府发布              | 实时（RSS订阅） |
| 大厅信息 | 政务办                | 季度更新        |
| 常见问题 | 12345热线工单（脱敏） | 月度导入        |

**维护责任**：初期由项目组人工整理，后期建立"AI训练师"岗位，对接各局联络员。

## 8. 里程碑计划

| 阶段       | 时间    | 交付物                                      |
| :--------- | :------ | :------------------------------------------ |
| **阶段一** | 第1-2周 | 搭建RAG流水线，录入50个高频问答，准确率测试 |
| **阶段二** | 第3-4周 | 实现意图识别+多轮追问，内测版上线           |
| **阶段三** | 第5-6周 | 材料预审+导航功能，对接1个政务大厅          |
| **阶段四** | 第7-8周 | 公测，收集反馈，优化准确率至95%             |
| **上线**   | 第9周   | 正式接入微信小程序/公众号                   |

## 9. 风险与应对

| 风险             | 可能性 | 应对措施                          |
| :--------------- | :----- | :-------------------------------- |
| **知识库不完整** | 高     | 初期聚焦TOP50问题，逐步扩展       |
| **AI产生幻觉**   | 中     | 强制RAG检索+来源引用，人工抽检    |
| **数据安全合规** | 中     | 本地部署，不调用云端API，日志审计 |
| **用户期望过高** | 高     | 明确告知AI能力边界，兜底转人工    |
| **部门配合度低** | 中     | 由政务办发文，建立知识库更新机制  |
